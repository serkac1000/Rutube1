About This Testing Procedure

This testing procedure provides a systematic approach to verify the correct functionality of your TeachableMachine implementation after migrating from PersonalImageClassifier.

Follow these steps in order to ensure that each component of your implementation is working correctly.

Pre-Testing Checklist

Before beginning the testing procedure, ensure you have:

  • Completed the migration from PersonalImageClassifier to TeachableMachine
  • Imported the TeachableMachine.aix extension
  • Created and exported a model from Teachable Machine
  • Updated all initialization code and event handlers
  • Added appropriate error handling
  • Verified all permissions are properly requested

Step-by-Step Testing Procedure

  1. Verify Extension Import

    First, ensure the TeachableMachine extension is properly imported:

    1. Open your project in MIT App Inventor
    2. Check the Components panel for TeachableMachine
    3. Verify that PersonalImageClassifier is no longer present

    Expected Outcome: TeachableMachine appears in your Components panel and is properly recognized.

  2. Test Camera Initialization

    Next, test that the Camera component initializes correctly:

    1. Add debug messages to your Camera initialization code
    2. Run the app and grant camera permissions when prompted
    3. Verify that the camera preview appears on screen

    Expected Outcome: Camera preview is visible and functioning properly.

  3. Test Model Loading

    Verify that the TeachableMachine model loads correctly:

    1. Add status messages for model loading steps
    2. Run the app and initiate model loading
    3. Check for successful loading messages
    4. Verify no errors appear in the logs

    Expected Outcome: Model loads successfully with no errors.

    Troubleshooting: If model fails to load, verify your internet connection and model URL.

  4. Test Classification Initialization

    Verify that the classifier starts properly:

    1. Add logging for the StartClassifier method call
    2. Run the app and start the classifier
    3. Check for any error messages

    Expected Outcome: Classifier starts without errors.

  5. Test Classification Results

    Verify that the classifier correctly identifies objects:

    1. Point the camera at objects from your training classes
    2. Verify that the GotPrediction event fires
    3. Check that the correct class is identified
    4. Verify confidence values are reasonable

    Expected Outcome: Objects are correctly classified with reasonable confidence.

  6. Test Application Logic

    Verify that your application correctly uses classification results:

    1. Test any conditional logic based on class identification
    2. Verify UI updates correctly based on classifications
    3. Test any actions triggered by specific classifications

    Expected Outcome: Application behaves as expected based on classification results.

  7. Test Error Handling

    Verify that your application handles errors gracefully:

    1. Test with airplane mode to simulate network issues
    2. Test with invalid model URL
    3. Verify appropriate error messages are displayed
    4. Check that the application recovers gracefully

    Expected Outcome: Application displays helpful error messages and doesn't crash.

  8. Performance Testing

    Evaluate the performance of your application:

    1. Observe classification speed and responsiveness
    2. Check memory usage over time
    3. Test battery impact during extended use

    Expected Outcome: Application performs efficiently without significant lag or battery drain.

Benchmarking

To evaluate the performance difference between PersonalImageClassifier and TeachableMachine, perform these benchmark tests:

Metric Test Method Expected Results
Initialization Time Measure time from StartClassifier to first classification TeachableMachine may be slower due to model downloading
Classification Speed Count classifications per minute Should be comparable between extensions
Classification Accuracy Test with controlled objects, calculate % correct Depends on model quality, but should be comparable
Memory Usage Monitor app memory in developer options TeachableMachine may use more memory
Battery Impact Run for 30 minutes, note battery percentage drop Should be comparable between extensions

Final Verification

After completing all tests, your app should:

  • Successfully load and use the TeachableMachine model
  • Correctly classify objects from your training classes
  • Display classification results appropriately
  • Handle errors gracefully
  • Perform efficiently on target devices

If all these criteria are met, your migration from PersonalImageClassifier to TeachableMachine.aix can be considered successful!
